# macOS desktop tasks + verifiers + action space — for Yutori review

A walkthrough of (1) the action space we expose to n1.5, (2) the design philosophy and shape of our verifiers, and (3) the 10 tasks with their exact instructions and grading commands.

## 1. Action space

We disable n1.5's browser-navigation tools and pass through everything else from `browser_tools_core-20260403`. The remaining 14 actions get mapped to the use.computer macOS-sandbox SDK. n1.5's coordinates are in a normalized 1000×1000 grid; we upscale them to the sandbox's real pixel space (≈1920×1080) before dispatch.

**Request config:**
```py
client.chat.completions.create(
    model="n1.5-latest",
    messages=...,
    max_completion_tokens=4096,
    extra_body={
        "tool_set": "browser_tools_core-20260403",
        "disable_tools": ["goto_url", "refresh", "go_back", "go_forward"],
    },
)
```

**Action mapping (n1.5 → use.computer SDK):**

| n1.5 action | SDK call | Coord transform | Notes |
|---|---|---|---|
| `left_click` | `mouse.click(x, y, button="left")` | upscale | `modifier` arg currently ignored |
| `right_click` | `mouse.click(x, y, button="right")` | upscale | |
| `middle_click` | `mouse.click(x, y, button="middle")` | upscale | |
| `double_click` | `mouse.click(x, y, button="left", double=True)` | upscale | |
| `triple_click` | 3× `mouse.click(...)` | upscale | |
| `scroll` | `mouse.scroll(x, y, direction, amount)` | upscale | `direction` ∈ {up,down,left,right}, `amount` passed through |
| `type` | `keyboard.type(text)` | — | |
| `key_press` | `keyboard.hotkey(key)` | — | `+`-joined hotkey string |
| `drag` | `mouse.drag(sx, sy, ex, ey)` | upscale both | |
| `mouse_move` | `mouse.move(x, y)` | upscale | |
| `wait` | `time.sleep(duration)` | — | |
| `hold_key` | best-effort single hotkey press | — | SDK has no held-key primitive |
| `mouse_down` / `mouse_up` | refused (logged) | — | SDK has no primitive |
| `goto_url` / `refresh` / `go_back` / `go_forward` | refused ("browser-only") | — | Also in `disable_tools` |

**Per-turn screenshot protocol:** before each predict, we append a fresh PNG screenshot as an `image_url` data-URL to the last user/tool message's content list. The most recent 3 screenshots are kept in context; older ones are stripped to bound payload size. Tool results are text-only (`{role:"tool", tool_call_id, content:[{type:"text", text:dmsg}]}`); the screenshot showing the post-action state gets attached to that tool message on the next predict.

**Termination:** assistant turn with empty `tool_calls` = task complete. We never see ` ```DONE``` ` (Anthropic-style).

## 2. Verifier design

### Philosophy

We follow Jason Wei's "asymmetry of verification" framing: maximize solve-side friction (UI depth, format precision, multi-app sequencing) while keeping verifiers cheap and deterministic. Every verifier in this set is a **single shell pipeline** that runs over SSH on the macOS sandbox and prints `True` or `False` to stdout. The runner reads stdout and looks for `"true"` (case-insensitive substring match) to score 100; anything else is 0.

Concretely each verifier satisfies five properties:

1. **Objective truth** — deterministic ground truth (string match, file content match, integer compare).
2. **Fast** — completes in <2s after a settle delay.
3. **Scalable** — pure stdout, parallelizable, no shared state between checks.
4. **Low-noise** — exact `grep -q` / `cmp -s` / `[ "$X" = "Y" ]` comparisons; no fuzzy matching, no LLM-as-judge, no screenshot-diff.
5. **Reproducible** — values either fixed in the task (e.g. `"June 15, 2026"`) or derived at grading time from authoritative sources (`sw_vers`, `sysctl`, `system_profiler`, `defaults read`).

### Verifier ingredients

We use a fixed toolkit:

- **`defaults read <domain> <key>`** — read a macOS preference plist. Most reliable for System Settings tasks. We sometimes prepend `killall cfprefsd; sleep 2` because the Settings UI doesn't always flush prefs to disk immediately.
- **`osascript -e 'tell application "X" to get ...'`** — read application state via AppleScript (Notes body, Reminders due date, Calendar event start date, Mail Drafts subject). Locale-dependent date strings — we anchor on substrings like `"June 15, 2026"` and `"9:00:00 AM"`.
- **File predicates** — `[ -f <path> ]`, `[ -d <path> ]`, `cmp -s <expected> <actual>` for byte-exact file comparison, `grep -qE` for line-pattern checks.
- **Dynamic derivation** — `sw_vers -productVersion`, `sysctl -n hw.memsize`, `sysctl -n machdep.cpu.brand_string`, `system_profiler SPHardwareDataType` for system-state values the model must transcribe into a note.

### Runner constraint to know about

The runner only evaluates `grading_command` entries whose value is `100` and stops at the first `True`. Partial-credit pills (50/10/etc.) are not supported in our current runner — so each task has exactly one verifier line, AND'd together with `&&`. If you wanted to grant partial credit you'd express it as multiple parallel 100-pt verifiers (any one matching = 100).

### Task JSON schema

Each task is a single JSON file under `infra/cli/tasks/<category>/<id>.json`:

```jsonc
{
  "id": "<uuid>",
  "snapshot": {"en": "snapshot_used_en"},   // which sandbox snapshot to restore
  "force_snapshot_recovery": true,           // hard-restore between tasks
  "task": {"en": "<instruction text>"},
  "pre_command": "<shell setup that runs on sandbox before agent starts>",
  "before_action_delay_seconds": 10,         // wait after pre_command before first screenshot
  "before_grading_delay_seconds": 30,        // wait after agent terminates before grading
  "grading_command": [["<single shell pipeline>", 100]]
}
```

### T1 · `advanced/746f816a` · save a shell script

**Instruction:**
> Create a shell script at `~/Documents/today.sh` that prints today's date in `YYYY-MM-DD` format and nothing else. Running `bash ~/Documents/today.sh` should output exactly one line matching `YYYY-MM-DD` (e.g. `2026-05-27`). You can use TextEdit (Plain Text mode) or any editor to create the file.

**pre_command:** `rm -f /Users/ec2-user/Documents/today.sh`

**grading_command:**
```sh
[ -f /Users/ec2-user/Documents/today.sh ] \
  && OUT=$(bash /Users/ec2-user/Documents/today.sh 2>/dev/null) \
  && echo "$OUT" | grep -qE '^[0-9]{4}-[0-9]{2}-[0-9]{2}$' \
  && [ "$OUT" = "$(date +%Y-%m-%d)" ] \
  && echo True || echo False
```

Solve = save dialog handling + plain-text mode + writing valid shell. Verify = run the script and `grep` its output.

### T2 · `multi_apps/46deeb3d` · Notes inventory (5 fields across 3 panes)

**Instruction:**
> Create a new note in Notes titled exactly `Device Inventory`. The body must contain exactly these FIVE lines, in this order, with the placeholders replaced by the current values from System Settings → General → About (and the Storage and Hardware sub-pages):
> ```
> macOS Version: <X.Y.Z>
> Memory: <NN GB>
> Chip: <chip name>
> Serial Number: <serial>
> Storage Available: <NN GB>
> ```
> For Storage Available, use the value shown in System Settings → General → Storage (e.g. `123 GB`). For Serial Number, click `More Info…` on the About page (or check `System Information` from the Apple menu).

**pre_command:** delete every note + delete every note in "Recently Deleted" + quit Notes; sleep 1.

**grading_command:**
```sh
M=$(sw_vers -productVersion)
R=$(( $(sysctl -n hw.memsize) / 1073741824 ))
C=$(sysctl -n machdep.cpu.brand_string)
SN=$(system_profiler SPHardwareDataType 2>/dev/null | awk -F': ' '/Serial Number/{print $2; exit}')
BODY=$(osascript -e 'tell application "Notes" to get body of every note' 2>/dev/null)
echo "$BODY" | grep -q "macOS Version: $M" \
  && echo "$BODY" | grep -q "Memory: ${R} GB" \
  && echo "$BODY" | grep -qF "Chip: $C" \
  && echo "$BODY" | grep -qF "Serial Number: $SN" \
  && echo "$BODY" | grep -qE 'Storage Available: [0-9]+ GB' \
  && echo True || echo False
```

Solve = read 5 system values across 3 settings panes, transcribe verbatim. Verify = compute expected values at grading time from authoritative sources, then `grep` the Notes body for each.

### T3 · `sys_and_interface/547c58f6` · Dock triple toggle

**Instruction:**
> In System Settings → Desktop & Dock (also called Dock & Menu Bar), apply ALL THREE of the following changes: (a) turn ON `Automatically hide and show the Dock`; (b) set `Position on screen` to `Left`; (c) drag the `Size` slider all the way to the left (smallest).

**pre_command:** `defaults write` to set autohide=false, orientation=bottom, tilesize=48; `killall Dock`; sleep 2.

**grading_command:**
```sh
killall cfprefsd 2>/dev/null; sleep 2
A=$(defaults read com.apple.dock autohide 2>/dev/null)
O=$(defaults read com.apple.dock orientation 2>/dev/null)
T=$(defaults read com.apple.dock tilesize 2>/dev/null)
[ "$A" = "1" ] && [ "$O" = "left" ] \
  && awk -v t="$T" 'BEGIN{exit !(t+0<=20)}' \
  && echo True || echo False
```

Solve = 3 changes in one pane, including a slider drag to minimum. Verify = `defaults read` on three keys.

  && echo "$MAIL" | grep -q "Standup" \
